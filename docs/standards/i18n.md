# i18n (NO/EN)

## Konfigurasjon

Astro i18n routing er konfigurert i `astro.config.mjs`:

```javascript
i18n: {
  defaultLocale: 'no',
  locales: ['no', 'en'],
  routing: {
    prefixDefaultLocale: true,
    fallbackType: 'redirect'
  }
}
```

## URL-struktur

- Norsk: `/no/...`
- Engelsk: `/en/...`

`/no` er default og brukes i URL-prefix.

## Sider

Hver side i `src/pages/[lang]/` er tilgjengelig på begge språk via dynamic routing.

## Oversettelser

- **Filer**: `src/translations/no.json` og `src/translations/en.json`
- **Format**: Nested JSON med dotted keys
- **Type-safety**: TypeScript types genereres automatisk fra `no.json`

### Translation keys med type-safety

Alle translation keys er type-safe via `TranslationKey` type:

```typescript
import { useTranslations } from "../utils/useTranslations";

const { t } = useTranslations(Astro.url);

// ✅ Type-safe - auto-completion i IDE
t("nav.helmaraton")
t("hero.title")
t("footer.contact")

// ❌ Type error hvis key ikke finnes
t("nav.invalid") // Compile error!
```

### Eksempler

```typescript
// Simple key
t("nav.about") // => "Om oss"

// Nested key
t("contact.people.livBentzen.name") // => "Liv Bentzen"

// Med fallback
t("missing.key", "Default text") // => "Default text"

// Hent rå data (arrays/objects)
getRaw("helgelandslopene.races") // => Array av løp
```

## useTranslations()

Hook som gir type-safe oversettelser:

```typescript
const { t, getRaw, currentLang } = useTranslations(Astro.url);
```

- **`t(key, fallback?)`**: Type-safe translation lookup
- **`getRaw(key)`**: Hent rå data (for arrays/objects)
- **`currentLang`**: Nåværende språk ("no" eller "en")

### Språkdeteksjon

Språk detekteres automatisk fra URL:
- `/en/*` → `en`
- Alt annet → `no` (default)

## Fallback-logikk

Hvis en engelsk nøkkel mangler:
1. Funksjonen returnerer `fallback` parameter hvis oppgitt
2. Ellers returnerer `undefined`
3. Norsk brukes som master - alle keys må finnes der

## Type-system

### Type-definisjon

`src/types/translations.ts` inneholder:

```typescript
type DotNestedKeys<T> // Utility type for dot-notation
type TranslationKey    // Auto-generert fra no.json
type Translations      // Full oversettelsesstruktur
```

### Auto-completion

IDE gir automatisk auto-completion for translation keys:
- IntelliSense viser alle tilgjengelige keys
- Type errors hvis du bruker ugyldig key
- Refactoring-safe - rename keys oppdaterer bruken

## Scripts

Hjelpescripts for oversettelsesarbeid:

```bash
npm run i18n:audit    # Sjekk for manglende oversettelser
npm run i18n:extract  # Trekk ut keys fra kode
npm run i18n:merge    # Slå sammen oversettelser
npm run i18n:fill     # Fyll inn manglende engelsk med norsk
```

## Anbefalt praksis

### DO ✅

- **Bruk stabile keys** - ikke oversett keys
  ```json
  { "nav.about": "Om oss" }  // ✅ Good
  { "nav.omOss": "Om oss" }  // ❌ Bad
  ```

- **Organiser hierarkisk** - bruk meningsfulle prefixer
  ```json
  {
    "nav": { ... },
    "hero": { ... },
    "contact": { ... }
  }
  ```

- **Hold keys konsis** - samme prefix for samme seksjon
  ```json
  "contact.email": "...",
  "contact.phone": "...",
  "contact.address": "..."
  ```

- **Bruk fallback** - alltid ha en backup-tekst
  ```typescript
  t("new.key", "Default text")
  ```

### DON'T ❌

- **Ikke hardkod tekst** - alt skal i translation files
  ```astro
  <h1>Om oss</h1>          // ❌ Bad
  <h1>{t("nav.about")}</h1> // ✅ Good
  ```

- **Ikke legg store HTML-blokker** - del opp i mindre keys
  ```json
  // ❌ Bad
  { "about.text": "<p>Lang tekst...</p><h2>...</h2>..." }

  // ✅ Good
  {
    "about.title": "Tittel",
    "about.intro": "Intro tekst",
    "about.body": "Brødtekst"
  }
  ```

- **Ikke dupliser keys** - bruk gjenbrukbare keys
  ```json
  // ❌ Bad
  { "button.submit": "Send", "form.send": "Send" }

  // ✅ Good
  { "common.submit": "Send" }
  ```

## Legge til nytt språk

For å legge til f.eks. svensk:

1. **Oppdater config**:
   ```javascript
   // astro.config.mjs
   i18n: {
     locales: ['no', 'en', 'sv'],
     // ...
   }
   ```

2. **Lag oversettingsfil**:
   ```bash
   cp src/translations/no.json src/translations/sv.json
   ```

3. **Oppdater useTranslations**:
   ```typescript
   const jsonTranslations =
     urlLang === "sv" ? svTranslations :
     urlLang === "en" ? enTranslations :
     noTranslations;
   ```

4. **Oversett innholdet i `sv.json``

## ⚠️ Common Pitfalls (vanlige feil)

### 1. Hardkodet tekst i komponenter

**❌ Feil:**
```astro
<h1>Om oss</h1>
<p>Velkommen til Torghatten Maraton!</p>
```

**✅ Riktig:**
```astro
<h1>{t("nav.about")}</h1>
<p>{t("hero.welcome")}</p>
```

### 2. Glemmer å oppdatere begge språkfiler

**Problem:** Legger til nøkkel i `no.json` men glemmer `en.json`

**Løsning:**
```bash
npm run i18n:fill  # Fyller manglende engelsk med norsk
npm run i18n:audit # Viser hva som mangler
```

### 3. Bruker norske nøkler

**❌ Feil:**
```json
{ "nav.omOss": "Om oss" }
```

**✅ Riktig:**
```json
{ "nav.about": "Om oss" }
```

**Hvorfor:** Nøkler skal være stabile og språk-agnostiske

### 4. Store HTML-blokker i oversettelser

**❌ Feil:**
```json
{
  "about.content": "<h2>Tittel</h2><p>Lang paragraf...</p><p>Enda mer...</p>"
}
```

**✅ Riktig:**
```json
{
  "about.title": "Tittel",
  "about.intro": "Lang paragraf...",
  "about.body": "Enda mer..."
}
```

### 5. Glemmer fallback i t()

**❌ Risikabelt:**
```typescript
t("new.experimental.key")  // Returnerer undefined hvis nøkkel mangler
```

**✅ Tryggere:**
```typescript
t("new.experimental.key", "Default text")  // Har backup
```

### 6. Ikke tester begge språk

**Problem:** Tester kun på NO, engelsk fungerer ikke

**Løsning:** Test alltid begge språk manuelt:
- Klikk gjennom `/no/*`
- Klikk gjennom `/en/*`
- Verifiser at språk-switcher fungerer

## 🧪 Testing Strategies

### Manual Testing

**1. Språk-switching test:**
```
1. Gå til /no/helmaraton
2. Klikk språk-switcher
3. Verifiser at du er på /en/helmaraton
4. Verifiser at alt tekst er engelsk
5. Klikk tilbake til norsk
6. Verifiser at du er tilbake på /no/helmaraton
```

**2. Complete language coverage test:**
```bash
# Sjekk at alle sider finnes på begge språk
npm run dev

# Besøk hver side på NO
/no/
/no/helmaraton
/no/halvmaraton
/no/gallery
# ...etc

# Besøk hver side på EN
/en/
/en/helmaraton
/en/halvmaraton
/en/gallery
# ...etc
```

### Automated Testing

**1. Type-checking (fanger manglende nøkler):**
```bash
npm run check
```

**2. Build test (fanger runtime-feil):**
```bash
npm run build
```

**3. Translation audit:**
```bash
npm run i18n:audit
```

### Testing Checklist

Før deploy, sjekk:

- [ ] `npm run check` kjører uten feil
- [ ] `npm run build` kjører uten feil
- [ ] `npm run i18n:audit` viser ingen manglende nøkler
- [ ] Manuell test av begge språk fullført
- [ ] Språk-switcher fungerer på alle sider
- [ ] Ingen hardkodet tekst i nye komponenter
- [ ] Alle nye nøkler følger navnekonvensjoner

### Browser DevTools Tips

**Finn hardkodet norsk tekst:**
1. Åpne `/en/` i nettleseren
2. Åpne DevTools Console
3. Kjør: `Array.from(document.body.querySelectorAll('*')).map(el => el.textContent).filter(text => text.match(/[æøå]/i))`
4. Hvis du ser norske ord på engelsk side → hardkodet tekst funnet!

**Verifiser translation keys brukes:**
1. Åpne DevTools Network tab
2. Refresh siden
3. Sjekk at `translations/*.json` lastes
4. Verifiser at riktig språkfil lastes basert på URL
